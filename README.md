# تقرير مشروع: مولّد مواقع من فكرة (Idea to Site)

## 1. الملخص التنفيذي

هذا المشروع عبارة عن **تطبيق ويب ثابت** يعمل بالكامل في متصفح المستخدم. يتيح للمستخدم كتابة **فكرة موقع** بلغة طبيعية (حالياً الواجهة بالإنجليزية)، ثم الضغط على زر **Generate** للانتقال إلى صفحة ثانية تعرض **معاينة حية** لموقع جاهز يشبه نوع الفكرة، مع إمكانية تغيير **عرض الشاشة** (عريض / متوسط / ضيق) وعرض **شفرة HTML و CSS و JavaScript** للقالب المختار.

**لا يوجد ذكاء اصطناعي ولا خادم خلفي:** التوليد يتم باختيار **قالب جاهز** مسبقاً بناءً على **مطابقة كلمات مفتاحية** في النص، مع بطاقات «أفكار سريعة» لاختيار قالب مباشر.

---

## 2. الفكرة والهدف

| العنصر | الشرح |
|--------|--------|
| **المشكلة** | رغبة المستخدم برؤية نسخة أولية سريعة لموقع يعكس فكرته (مثل: محاسبة، محفظة، مطعم) دون برمجة معقدة أو خدمات سحابية. |
| **الحل** | مكتبة **قوالب HTML/CSS/JS** ثابتة + **محرك مطابقة بسيط** يختار الأنسب حسب الكلمات في وصف المستخدم. |
| **الجمهور** | من يريد نموذجاً بصرياً سريعاً، أو تعلم بنية صفحات جاهزة، أو عرض فكرة لعميل. |

---

## 3. الميزات الرئيسية

1. **صفحة رئيسية (`index.html`)**  
   - حقل نصي لوصف الفكرة.  
   - زر **Generate site** للانتقال للمعاينة.  
   - بطاقات **Quick-start ideas**؛ كل بطاقة تملأ مثالاً نصياً وتربط قالباً محدداً حتى يضغط المستخدم Generate.

2. **صفحة المعاينة (`preview.html`)** — أسلوب يشبه أدوات مثل Lovable من حيث الفصل بين لوحة جانبية ومعاينة:  
   - عرض **نص الفكرة** واسم **القالب** المستخدم.  
   - تنبيه عند **عدم وجود تطابق قوي** للكلمات (يُعرض قالب عام).  
   - **معاينة داخل iframe** لموقع القالب كاملاً.  
   - أزرار **عرض المعاينة**: Large (~1280px)، Medium (~768px)، Small (~375px).  
   - تبويبات **HTML / CSS / JavaScript** لعرض الشفرة، مع زر **Copy**.

3. **العملة في القوالب المالية**  
   - قوالب المحاسبة والمحفظة تستخدم **الريال العُماني (OMR)** بصيغة مناسبة للعرض (بما في ذلك `toLocaleString` مع `en-OM` حيث ينطبق).

4. **واجهة المستخدم**  
   - اللغة الحالية للواجهة والقوالب المعروضة: **الإنجليزية**، اتجاه الصفحة **LTR**.

---

## 4. التقنيات المستخدمة

| التقنية | الاستخدام |
|---------|-----------|
| **HTML5** | هيكل الصفحات والمعاينة داخل iframe. |
| **CSS3** | تنسيق الصفحات، الشبكة، الألوان، أحجام المعاينة. |
| **JavaScript (Vanilla)** | المطابقة، التخزين المؤقت، بناء مستند المعاينة، التبويبات، النسخ. |

**غير مستخدم:** React، Vue، Node، قواعد بيانات، APIs خارجية، بناء (build) أو حزم npm.

---

## 5. هيكل المشروع

```
falahp2/
├── index.html          # الصفحة الأولى: إدخال الفكرة والتوليد
├── preview.html        # الصفحة الثانية: المعاينة + الشفرة + أحجام الشاشة
├── README.md           # هذا التقرير (بالعربية)
├── css/
│   ├── main.css        # أنماط الصفحة الرئيسية
│   └── preview.css     # أنماط صفحة المعاينة والشريط الجانبي
└── js/
    ├── templates.js    # تعريف كل القوالب (html, css, js, keywords, title)
    ├── matcher.js      # تطبيع النص + حساب النقاط + قالب افتراضي
    ├── app.js          # منطق الصفحة الرئيسية والبطاقات والانتقال
    └── preview.js      # تحميل القالب، iframe، التبويبات، أحجام العرض
```

## 6. تسلسل استخدام الموقع (تدفق المستخدم)

```
المستخدم يفتح index.html
        ↓
يكتب فكرته و/أو يختار بطاقة سريعة
        ↓
يضغط "Generate site"
        ↓
يُحفظ الاختيار (انظر القسم التالي) ويُفتح preview.html
        ↓
يظهر الموقع الجاهز في iframe + الشفرة في التبويبات
        ↓
يمكن تغيير عرض المعاينة ونسخ الكود
        ↓
"Edit idea" للعودة إلى index.html
```

---

## 7. آلية التوليد والمطابقة

1. **تطبيع النص** (`matcher.js`): إزالة مسافات زائدة، تحويل لأحرف صغيرة، تبسيط علامات الترقيم.  
2. **للقوالب غير الافتراضية:** لكل قالب قائمة `keywords`. إذا وُجدت كلمة المفتاح داخل نص المستخدم (كجزء من السلسلة)، تُحسب نقطة لذلك القالب.  
3. **الفائز:** القالب ذو **أعلى عدد من التطابقات**.  
4. **بدون أي تطابق:** يُستخدم قالب **`default`** (صفحة ترحيب عامة)، ويظهر تنبيه في صفحة المعاينة.  
5. **البطاقات السريعة:** عند النقر، يُحدَّد القالب مباشرة حتى يضغط المستخدم Generate؛ إذا عدّل المستخدم النص يدوياً بعد ذلك، يُلغى الربط المسبق ويُعاد الاعتماد على مطابقة النص فقط.

---

## 8. نقل البيانات بين الصفحتين

| الطريقة | الوصف |
|---------|--------|
| **`sessionStorage`** | يُفضَّل عند التشغيل عبر `http://localhost` لنفس الأصل (origin). |
| **معاملات الرابط (Query String)** | يُمرَّر `id` القالب، `m` (تطابق نعم/لا)، و`idea` (النص) في عنوان `preview.html?...` لضمان عمل التوليد حتى عند فتح الملفات محلياً (`file://`) حيث قد لا يُشارك `sessionStorage` بين ملفين. |

---

## 9. القوالب المتوفرة

| المعرف (`id`) | العنوان (بالإنجليزية في الكود) | فكرة الاستخدام |
|---------------|----------------------------------|----------------|
| `accounting` | Accounting & invoices | لوحة مالية، فواتير، ملخص أرقام (OMR) |
| `wallet` | Personal wallet | رصيد وحركات تجريبية (OMR) |
| `restaurant` | Restaurant menu | قائمة طعام مع تصفية حسب الفئة |
| `portfolio` | Portfolio | صفحة تعريف ومشاريع |
| `landing` | Product landing page | هبوط بريد واشتراك تجريبي |
| `default` | Generic welcome page | عند ضعف التطابق مع الكلمات |

كل قالب يحتوي على ثلاثة أجزاء نصية: **`html`** (محتوى داخل body)، **`css`**، **`js`** يُدمجون داخل مستند كامل يُحقن في iframe.

---

## 10. التصميم البصري

- **خلفية:** أبيض وفتحات رمادية فاتحة (`#fafafa`) للتمييز.  
- **أزرار أساسية:** بني فاتح تقريباً `#c9a882` مع حالة hover أغمق.  
- **نصوص داكنة:** درجات بني/رمادي للعناوين والوصف.

---

## 11. طريقة التشغيل المحلي

1. افتح الطرفية (Terminal) داخل مجلد المشروع `falahp2`.  
2. شغّل خادماً ثابتاً، مثلاً:  
   ```bash
   python3 -m http.server 8000
   ```  
3. في المتصفح افتح:  
   `http://127.0.0.1:8000/index.html`  
   أو `http://localhost:8000/index.html`

**ملاحظة:** فتح `index.html` مباشرة من القرص قد يعمل، لكن التوليد يعتمد أيضاً على معاملات الرابط؛ استخدام خادم محلي يقلل مشاكل المسارات والأصل.

---

## 12. القيود والاعتبارات

- التوليد **ليس إنشاءً حراً**؛ هو **اختيار من قائمة محدودة** من القوالب.  
- جودة المطابقة تعتمد على وجود **كلمات مفتاحية** مناسبة في `templates.js`.  
- **النسخ إلى الحافظة** يحتاج عادةً سياقاً آمناً (`https` أو localhost).  
- طول الرابط قد يُقتصر عند فكرة طول جداً (يُختصر النص في الرابط مع الإبقاء على التخزين المحلي عند توفره).

---

## 13. خاتمة التقرير

المشروع يحقق هدفاً واضحاً: **عرض سريع لموقع نموذجي** مبني على وصف نصي، باستخدام **HTML و CSS و JavaScript فقط**، **بدون باكند وبدون ذكاء اصطناعي**، مع تجربة معاينة قريبة من أدوات التطوير البصرية (لوحة جانبية + معاينة + شفرة). التقرير أعلاه يغطي البنية، السلوك، الملفات، والتشغيل ليكون مرجعاً كاملاً للموقع.
